# Aggregation Pattern

Student understands aggregation as combining multiple records into a summary. Implemented the find-or-create pattern: search for existing entry, update if found, push new entry if not.

## Evidence
- Solved "aggregate scores by person" using loop + find + conditional
- Correctly described the algorithm in plain English
- Learned terminology: "aggregation = combining multiple records into a summary"

## Implications
- Ready for multi-key aggregation (e.g., by exporter + grade)
- Reinforcement exercise: have student explain the pattern back before moving on
- This pattern recurs in many real-world problems — worth revisiting with different data
