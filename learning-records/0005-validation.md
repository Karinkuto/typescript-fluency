# Validation Pattern

Student can implement validation: loop through records, check rules, keep or discard. Understands that validation is a loop + conditional + keep/discard pattern.

## Evidence
- Implemented exporter-required and bags-greater-than-zero checks
- Correctly described the pattern in plain English

## Implications
- Ready for more complex validation (cross-field rules, error accumulation)
- Can teach filter() as a natural expression of this pattern
- Business rules vs implementation rules distinction was a breakthrough — keep reinforcing
