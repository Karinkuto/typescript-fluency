# Student Profile Summary

The student can reason algorithmically. The gap is in translating reasoning into executable code, not in the reasoning itself. Student mistakenly believes they "can't think algorithmically" — evidence contradicts this.

## Observed Capabilities
- Identifies inputs and outputs
- Describes algorithms in plain English
- Reasons about business rules and data integrity
- Decomposes problems

## Needs Practice
- Expressing logic in code (formal expression)
- Standard library familiarity (find, some, map, filter)
- Boolean logic
- TypeScript type narrowing

## Key Insight
Treat the student as someone building implementation fluency, not someone learning how to think. The distinction matters.
