# Boolean Logic Gap

Student initially wrote `value !== "" || null || undefined`, revealing incomplete understanding of boolean expressions. This is a known weak area that needs reinforcement.

## Evidence
- Incorrect boolean expression in code
- Did not understand short-circuit evaluation or how `||` works with non-boolean values

## Implications
- Needs explicit exercises on boolean logic
- Should teach: truthy/falsy values, short-circuit evaluation, correct ways to check for null/undefined
- This is a tooling familiarity issue, not a reasoning issue
