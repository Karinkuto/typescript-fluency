# Mission: Build Implementation Fluency in TypeScript

## Why
The student can reason about problems, decompose them, and describe algorithms in plain English — but struggles to translate that reasoning into executable code without AI assistance. The goal is to close that gap so they can implement solutions independently.

## Success looks like
- Can take a problem description and write working TypeScript code without AI help
- Can express filtering, aggregation, and data transformation logic in code
- Understands common array/object methods (find, some, map, filter, reduce) well enough to use them fluently
- Can read TypeScript errors (especially around undefined/null) and reason about what they mean

## Constraints
- Prioritize reasoning over syntax — always start with "what am I actually trying to do?"
- Teach through guided discovery, not answer-giving
- Keep exercises small and composable
- Reinforce boolean logic and type narrowing explicitly

## Out of scope
- Advanced topics like generics, decorators, or framework-specific patterns
- Backend architecture or system design
- Complex async patterns (for now)
